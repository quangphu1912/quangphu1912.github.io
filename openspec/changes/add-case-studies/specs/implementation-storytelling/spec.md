## ADDED Requirements

### Requirement: Implementation journey narrative

Each case study MUST tell the story of implementation from start to finish.

#### Scenario: Document timeline
- **WHEN** case study includes Implementation Journey
- **THEN** it MUST specify project duration (e.g., "6 weeks", "3 months")
- **AND** MUST break down into phases with timeframes
- **AND** MUST show progression from planning to deployment

#### Scenario: Document challenges and solutions
- **WHEN** case study describes implementation
- **THEN** it MUST describe at least 2-3 major challenges faced
- **AND** for each challenge, MUST explain how it was overcome
- **AND** MUST be specific about technical solutions (e.g., "switched from X to Y")

#### Scenario: Document iterations
- **WHEN** project went through multiple iterations
- **THEN** case study MUST describe what changed between iterations
- **AND** MUST explain why changes were made
- **AND** MUST show improvement in metrics after each iteration

### Requirement: Lessons learned

Each case study MUST include a Lessons Learned section with applicable insights.

#### Scenario: Document what worked
- **WHEN** case study includes Lessons Learned
- **THEN** it MUST list 2-3 approaches that worked well
- **AND** MUST explain why they worked
- **AND** MUST describe how they could be applied to other problems

#### Scenario: Document what to do differently
- **WHEN** case study includes Lessons Learned
- **THEN** it MUST list 1-2 things to do differently next time
- **AND** MUST explain why different approach would be better
- **AND** MUST be actionable for future projects

#### Scenario: Assess applicability
- **WHEN** lessons are documented
- **THEN** case study MUST specify when lessons apply (e.g., "For time-series projects", "When data is sparse")
- **AND** MUST specify when they don't apply
